# Cash Change From — Design

## Задача

При оплате наличными пользователь указывает сумму, с которой нужна сдача. Курьер видит это и приносит сдачу. Реализуем только сторону пользователя (фронтенд).

## Решения

- **UI**: Раскрывающееся поле внутри Bottom Sheet выбора способа оплаты
- **Валидация**: Сумма >= стоимости заказа, поле обязательное при наличных
- **Передача данных**: Дописываем в `comment` строку "Сдача с {N} ₽"
- **Бэкенд**: Без изменений

## Изменения

### Файл: `cart_screen.dart`

**Новая переменная:**
- `double? _changeFrom` — сумма сдачи (null если карта)

**Bottom Sheet (`_showPaymentMethodPicker`):**
1. Обернуть в `StatefulBuilder` для локального стейта
2. При выборе "Наличными" → анимированно показать поле ввода:
   - Текст: "Сдача с какой суммы?"
   - `TextField`, `keyboardType: TextInputType.number`, placeholder: "Например, 1000", суффикс "₽"
3. При переключении на "Картой курьеру" → поле скрывается
4. Кнопка "Готово" внизу:
   - Наличные: валидация суммы >= `finalTotal`, иначе ошибка "Сумма должна быть не менее {итого} ₽"
   - Карта: закрывает без валидации
5. При выборе карты `_changeFrom` сбрасывается в null

**Отображение на экране корзины:**
- Подзаголовок способа оплаты: "Наличными · Сдача с 1000 ₽" (если `_changeFrom != null`)

**Метод `_submitOrder`:**
- Если `_paymentMethod == 'cash'` и `_changeFrom != null`:
  - Комментарий = `"${userComment}\nСдача с ${_changeFrom!.toInt()} ₽".trim()`

### Файлы без изменений
- Бэкенд (модели, views, serializers)
- `OrderService.createOrder()` — сигнатура не меняется
- Модели (`OrderModel`, `OrderDetailModel`)
- Повтор заказа (`repeatOrder`) — не трогаем

## Объём
- 1 файл: `cart_screen.dart`
- ~50-70 строк нового кода
