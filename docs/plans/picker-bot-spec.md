# ТЗ: Telegram-бот сборщика (Picker Bot)

## 1. Назначение

Бот для сотрудников-сборщиков магазина. Позволяет принимать новые заказы, собирать их и передавать курьеру (доставка) или выдавать клиенту (самовывоз).

## 2. Место в системе

### Текущий flow:
```
new → assembly → ready → delivery → arrived → completed
```

### Новый flow:
```
Доставка:   new → accepted → assembly → ready → delivery → arrived → completed
                  ↑ picker_bot ──────────── ↑    ↑ courier_bot ────────────── ↑

Самовывоз:  new → accepted → assembly → ready → completed
                  ↑ picker_bot ────────────────────────── ↑
```

### Статусы и что видит клиент в приложении:

| Статус | Текст в приложении |
|--------|-------------------|
| `new` | Новый заказ |
| `accepted` | Заказ принят |
| `assembly` | Заказ собирается |
| `ready` + доставка | Заказ собран, ждёт курьера |
| `ready` + самовывоз | Ваш заказ готов, можете забрать |

## 3. Авторизация

Whitelist по Telegram ID через env-переменную `PICKER_ALLOWED_TG_IDS` (аналогично курьерскому боту).

## 4. Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие + проверка доступа |
| `/orders` | Новые заказы (статус `new`) |
| `/active` | Мои заказы в работе (`accepted` / `assembly` / `ready`+самовывоз) |
| `/completed` | Отчёт за сегодня (количество собранных) |
| `/help` | Справка по командам |

## 5. Сценарии работы

### 5.1. Просмотр новых заказов (`/orders`)

Inline-список заказов со статусом `new`:
```
📦 Новые заказы:

#42 🆕 Новый — 1500₽
#43 🆕 Новый — 890₽
```

При нажатии — карточка с деталями.

### 5.2. Карточка заказа

```
Заказ #42

📋 Состав:
  • Молоко 3.2% x2 — 120₽
  • Хлеб белый x1 — 45₽
  • Сыр Гауда x1 — 350₽

💰 Сумма товаров: 635₽
🚚 Доставка: 150₽
💵 Итого: 785₽
💳 Оплата: Картой курьеру
📦 Тип: Доставка
🏠 Адрес: ул. Ленина, 5
📞 Телефон: +7-999-123-45-67

Статус: Новый заказ
```

Кнопки зависят от статуса (см. 5.3–5.7).

### 5.3. Шаг 1: Принять заказ

Статус: `new`
Кнопка: **📋 Принять**

Действие:
- `new → accepted`
- `assembled_by` = Telegram ID сборщика
- Клиент в приложении видит: «Заказ принят»

### 5.4. Шаг 2: Начать сборку

Статус: `accepted`
Кнопки:
- **📦 Собираем** (accepted → assembly)
- **↩️ Вернуть заказ** (accepted → new, очистка `assembled_by`)
- **⬅ Назад**

Действие «Собираем»:
- `accepted → assembly`
- Клиент в приложении видит: «Заказ собирается»

### 5.5. Шаг 3: Завершить сборку

Статус: `assembly`
Кнопки:
- **✅ Заказ собрал** (assembly → ready)
- **↩️ Вернуть заказ** (assembly → new, очистка `assembled_by`)
- **⬅ Назад**

Действие «Заказ собрал»:
- `assembly → ready`
- **Доставка:** курьеры получают push (существующий сигнал `notify_couriers_new_order`)
- **Самовывоз:** клиент видит «Ваш заказ готов, можете забрать». В боте сборщика появляется кнопка «🤝 Клиент забрал»

### 5.6. Шаг 4: Выдача самовывоза

Статус: `ready` + `fulfillment_type = "pickup"`
Кнопка: **🤝 Клиент забрал** (ready → completed)

Действие:
- `ready → completed`
- Заказ завершён

### 5.7. Возврат заказа

Кнопка **↩️ Вернуть заказ** доступна на шагах 2 и 3 (`accepted`, `assembly`).
- Сбрасывает статус в `new`
- Очищает `assembled_by`
- Заказ возвращается в общий пул (`/orders`)

Кейсы: случайно взял, не успевает собрать.

### 5.8. Мои активные (`/active`)

Заказы, привязанные к текущему сборщику:
- Статус `accepted` или `assembly`
- Статус `ready` + `fulfillment_type = "pickup"` (ожидает выдачи)

### 5.9. Отчёт (`/completed`)

```
📋 Отчёт за сегодня:

📦 Собрано заказов: 12
```

Только количество, без денег (ставка за сборку не считается).

## 6. Уведомления

Celery-таска `notify_pickers_new_order` — отправляет push сборщикам при создании нового заказа (статус `new`). Аналогично `notify_couriers_new_order`.

## 7. Изменения в backend

### 7.1. Модель Order

Новый статус в `STATUS_CHOICES`:
```python
('accepted', 'Заказ принят'),
```

Полный список:
```python
STATUS_CHOICES = [
    ('new', 'Новый'),
    ('accepted', 'Заказ принят'),
    ('assembly', 'Заказ собирается'),
    ('ready', 'Заказ собран'),
    ('delivery', 'Курьер забрал заказ и в пути'),
    ('arrived', 'Курьер пришёл и ждёт вас'),
    ('completed', 'Заказ доставлен'),
    ('canceled', 'Отменён'),
]
```

Новое поле:
```python
assembled_by = models.BigIntegerField(
    null=True, blank=True, db_index=True,
    verbose_name="Telegram ID сборщика"
)
```

### 7.2. Новые bot_api эндпоинты

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/bot/orders/new/` | Заказы со статусом `new` |
| GET | `/api/bot/orders/my-active/` | Заказы сборщика (`accepted`/`assembly`/`ready`+pickup), фильтр по `assembled_by` |
| GET | `/api/bot/orders/assembled-today/` | Кол-во собранных за сегодня по `assembled_by` |

Смена статуса — через существующий `/onec/order/status`.

### 7.3. Сигналы

В `orders/signals.py`:
- При создании заказа (`new`) — Celery-таска `notify_pickers_new_order`

### 7.4. Эндпоинт `/onec/order/status`

Добавить `accepted` в список допустимых статусов (уже есть `allowed` set). Передача `assembled_by` при переходах.

### 7.5. Settings

Новые переменные:
- `PICKER_BOT_TOKEN`
- `PICKER_ALLOWED_TG_IDS`

## 10. Что НЕ входит в scope

- Частичная сборка (отметка отдельных позиций)
- Замена товаров при отсутствии
- Печать этикеток/накладных
- Фото собранного заказа
- Чат с клиентом
- Изменения в мобильном приложении (потом)

## 11. Зависимости

- `chat_cleanup.py` и `retry.py` дублируются из courier_bot → рефакторинг в `shared/bot_utils/` (отдельная задача, не блокирует)

## 12. Решённые вопросы

1. ✅ **Привязка сборщика** — да, поле `assembled_by`
2. ✅ **Перехват заказа** — возврат в общий пул кнопкой «↩️ Вернуть заказ»
3. ✅ **Ставка за сборку** — не считается, только количество
4. ✅ **Самовывоз** — сборщик видит, собирает и отмечает выдачу («🤝 Клиент забрал»)
5. ✅ **Новый статус `accepted`** — добавляется между `new` и `assembly`
6. ✅ **Flow для всех заказов** — picker_bot работает и с доставкой, и с самовывозом
7. ✅ **Scope** — только backend + бот, приложение потом
