# План v7: API бонусного статуса и рекламных кампаний для 1С + автопересчёт RFM

## Бизнес-уточнение: допустимость окна monthly batch

- Магазин работает максимум до 20:00, в летнее время до 23:00.
- Данные из 1С приходят на backend максимум через 30 минут.
- Агрегаты CustomUser обновляются синхронно в receipt endpoint (подтверждено аудитом кода).
- Поэтому даже последняя продажа дня (23:00) + лаг 1С (30 мин) = 23:30 — данные на backend до monthly batch в 00:05.
- Окно 00:05–00:10 1-го числа допустимо как операционный компромисс v1.
- При изменении режима работы (после полуночи, ночная доставка, 24/7) или увеличении лага 1С > 30 минут — решение подлежит пересмотру.

## Decision point: источник данных monthly batch — РЕШЁН

Вариант A подтверждён. Агрегаты CustomUser (`last_purchase_date`, `purchase_count`, `total_spent`) обновляются синхронно в receipt.py через F-expressions в `.update()`. Monthly batch использует их как источник истины.
