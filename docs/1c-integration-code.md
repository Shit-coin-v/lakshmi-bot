# Код 1С для интеграции с Lakshmi

> Обновлено: 2026-04-25

## Общий модуль: `ЛакшмиИнтеграция`

Все запросы к API Lakshmi используют:
- Заголовок `X-Api-Key` с ключом интеграции (`INTEGRATION_API_KEY` / `ONEC_API_KEY`)
- Формат JSON, кодировка UTF-8
- Base URL сервера (например `https://your-domain.tld`)
- Аутентификация выполняется декоратором `@require_onec_auth` (constant-time compare API-ключа + опциональный IP whitelist `ONEC_ALLOW_IPS`)

В production при пустом `ONEC_ALLOW_IPS` сервер отклоняет все запросы (403). Для `/onec/receipt` дополнительно требуется `X-Idempotency-Key` (UUID).

---

## 1. Общие функции

```bsl
// Общий модуль "ЛакшмиИнтеграция"

Перем БазовыйURL;   // "https://your-domain.tld"
Перем КлючAPI;       // значение ONEC_API_KEY / INTEGRATION_API_KEY

Процедура УстановитьНастройки() Экспорт
    БазовыйURL = Константы.ЛакшмиURL.Получить();
    КлючAPI     = Константы.ЛакшмиAPIKey.Получить();
КонецПроцедуры

// Выполнить HTTP-запрос к API Lakshmi
//
// Параметры:
//   Метод   - Строка - "GET" или "POST"
//   Путь    - Строка - путь endpoint, например "/onec/health"
//   Тело    - Строка - JSON-строка тела запроса (для POST), "" для GET
//   ДопЗаголовки - Соответствие - дополнительные заголовки (опционально)
//
// Возвращаемое значение:
//   Структура - КодОтвета (Число), Тело (Строка)
//
Функция ВыполнитьЗапрос(Метод, Путь, Тело = "", ДопЗаголовки = Неопределено) Экспорт

    УстановитьНастройки();

    СтруктураURL = РазобратьURL(БазовыйURL);
    Хост   = СтруктураURL.Хост;
    Порт   = СтруктураURL.Порт;
    Защита = СтруктураURL.Защита;

    Соединение = Новый HTTPСоединение(Хост, Порт,,,, 30, Защита);

    Запрос = Новый HTTPЗапрос(Путь);
    Запрос.Заголовки.Вставить("X-Api-Key", КлючAPI);
    Запрос.Заголовки.Вставить("Content-Type", "application/json");

    Если ДопЗаголовки <> Неопределено Тогда
        Для Каждого Заголовок Из ДопЗаголовки Цикл
            Запрос.Заголовки.Вставить(Заголовок.Ключ, Заголовок.Значение);
        КонецЦикла;
    КонецЕсли;

    Если НЕ ПустаяСтрока(Тело) Тогда
        Запрос.УстановитьТелоИзСтроки(Тело, КодировкаТекста.UTF8);
    КонецЕсли;

    Если ВРег(Метод) = "POST" Тогда
        Ответ = Соединение.ОтправитьДляОбработки(Запрос);
    Иначе
        Ответ = Соединение.Получить(Запрос);
    КонецЕсли;

    Результат = Новый Структура;
    Результат.Вставить("КодОтвета", Ответ.КодСостояния);
    Результат.Вставить("Тело", Ответ.ПолучитьТелоКакСтроку(КодировкаТекста.UTF8));

    Возврат Результат;

КонецФункции

Функция РазобратьURL(URL)
    Результат = Новый Структура("Хост, Порт, Защита");
    Если СтрНачинаетсяС(URL, "https://") Тогда
        Результат.Защита = Новый ЗащищенноеСоединениеOpenSSL();
        Результат.Порт = 443;
        Результат.Хост = СтрЗаменить(URL, "https://", "");
    Иначе
        Результат.Защита = Неопределено;
        Результат.Порт = 80;
        Результат.Хост = СтрЗаменить(URL, "http://", "");
    КонецЕсли;
    // Убрать trailing slash
    Если СтрЗаканчиваетсяНа(Результат.Хост, "/") Тогда
        Результат.Хост = Лев(Результат.Хост, СтрДлина(Результат.Хост) - 1);
    КонецЕсли;
    Возврат Результат;
КонецФункции

Функция РазобратьJSON(СтрокаJSON) Экспорт
    ЧтениеJSON = Новый ЧтениеJSON;
    ЧтениеJSON.УстановитьСтроку(СтрокаJSON);
    Возврат ПрочитатьJSON(ЧтениеJSON);
КонецФункции

Функция СформироватьJSON(Данные) Экспорт
    ЗаписьJSON = Новый ЗаписьJSON;
    ЗаписьJSON.УстановитьСтроку();
    ЗаписатьJSON(ЗаписьJSON, Данные);
    Возврат ЗаписьJSON.Закрыть();
КонецФункции

Функция НовыйUUID() Экспорт
    Возврат Строка(Новый УникальныйИдентификатор);
КонецФункции
```

## 3. Загрузка чека (Receipt)

`POST /onec/receipt` — отправка чека продажи в Lakshmi для начисления/списания бонусов.

Особенности:
- Идентификация клиента — по полю `customer.card_id` (не `telegram_id`/`one_c_guid`).
- Обязателен заголовок `X-Idempotency-Key` (UUID). Дубль по `receipt_guid` возвращает `200`/`already exists`.
- Поддерживается `purchase_type ∈ {"in_store", "delivery", "pickup"}` (по умолчанию `in_store`).
- Запрещены дублирующиеся `line_number` в одном чеке (409 `duplicate_receipt_line`).
- Если `customer.card_id` пуст или клиент не зарегистрирован — чек привязывается к гостевому пользователю (если включён `GUEST_TELEGRAM_ID`).

```bsl
// Параметры:
//   ДанныеЧека - Структура:
//     .GUIDЧека         - Строка - уникальный GUID чека (receipt_guid)
//     .Дата              - Дата
//     .КодМагазина       - Строка
//     .CardID            - Строка - номер бонусной карты клиента (пусто = гость)
//     .ТипПокупки        - Строка - "in_store" / "delivery" / "pickup"
//     .Позиции           - Массив структур:
//       .КодТовара         - Строка
//       .Наименование      - Строка
//       .Количество        - Число (>= 0.001, до 3 знаков)
//       .Цена              - Число (>= 0)
//       .СуммаСкидки       - Число
//       .Акционный         - Булево
//       .НомерСтроки       - Число (уникальный в пределах чека, >= 0)
//       .НачисленоБонусов  - Число (>= 0, опционально)
//       .Категория         - Строка (опционально)
//     .СуммаИтого         - Число
//     .СуммаСкидок        - Число
//     .СписаноБонусов     - Число
//     .НачисленоБонусов   - Число
//
Функция ОтправитьЧек(ДанныеЧека) Экспорт

    Позиции = Новый Массив;
    Для Каждого Строка Из ДанныеЧека.Позиции Цикл
        Позиция = Новый Структура;
        Позиция.Вставить("product_code",     Строка.КодТовара);
        Позиция.Вставить("name",             Строка.Наименование);
        Позиция.Вставить("quantity",         Формат(Строка.Количество, "ЧРД=.; ЧН=0; ЧГ="));
        Позиция.Вставить("price",            Формат(Строка.Цена, "ЧРД=.; ЧН=0.00; ЧГ="));
        Позиция.Вставить("discount_amount",  Формат(Строка.СуммаСкидки, "ЧРД=.; ЧН=0.00; ЧГ="));
        Позиция.Вставить("is_promotional",   Строка.Акционный);
        Позиция.Вставить("line_number",      Строка.НомерСтроки);
        Позиция.Вставить("bonus_earned",     Формат(Строка.НачисленоБонусов, "ЧРД=.; ЧН=0.00; ЧГ="));
        Если ЗначениеЗаполнено(Строка.Категория) Тогда
            Позиция.Вставить("category",     Строка.Категория);
        КонецЕсли;
        Позиции.Добавить(Позиция);
    КонецЦикла;

    Клиент = Новый Структура;
    Клиент.Вставить("card_id", ДанныеЧека.CardID);

    Итоги = Новый Структура;
    Итоги.Вставить("total_amount",    Формат(ДанныеЧека.СуммаИтого, "ЧРД=.; ЧН=0.00; ЧГ="));
    Итоги.Вставить("discount_total",  Формат(ДанныеЧека.СуммаСкидок, "ЧРД=.; ЧН=0.00; ЧГ="));
    Итоги.Вставить("bonus_spent",     Формат(ДанныеЧека.СписаноБонусов, "ЧРД=.; ЧН=0.00; ЧГ="));
    Итоги.Вставить("bonus_earned",    Формат(ДанныеЧека.НачисленоБонусов, "ЧРД=.; ЧН=0.00; ЧГ="));

    ТелоЗапроса = Новый Структура;
    ТелоЗапроса.Вставить("receipt_guid",  ДанныеЧека.GUIDЧека);
    ТелоЗапроса.Вставить("datetime",      Формат(ДанныеЧека.Дата, "ДФ=yyyy-MM-ddTHH:mm:ss"));
    ТелоЗапроса.Вставить("store_id",      ДанныеЧека.КодМагазина);
    ТелоЗапроса.Вставить("customer",      Клиент);
    ТелоЗапроса.Вставить("positions",     Позиции);
    ТелоЗапроса.Вставить("totals",        Итоги);
    ТелоЗапроса.Вставить("purchase_type", ДанныеЧека.ТипПокупки);

    ДопЗаголовки = Новый Соответствие;
    ДопЗаголовки.Вставить("X-Idempotency-Key", НовыйUUID());

    Попытка
        Результат = ВыполнитьЗапрос("POST", "/onec/receipt", СформироватьJSON(ТелоЗапроса), ДопЗаголовки);
    Исключение
        Сообщить("Ошибка отправки чека: " + ОписаниеОшибки());
        Возврат Неопределено;
    КонецПопытки;

    Если Результат.КодОтвета = 200 ИЛИ Результат.КодОтвета = 201 Тогда
        Возврат РазобратьJSON(Результат.Тело);
    Иначе
        Сообщить("Ошибка отправки чека: HTTP " + Результат.КодОтвета + " — " + Результат.Тело);
        Возврат Неопределено;
    КонецЕсли;

КонецФункции
```

Ответ (200/201):
```json
{
  "status": "ok",
  "receipt_guid": "...",
  "created_count": 3,
  "allocations": [
    {"product_code": "SKU1", "quantity": 2.0, "total_amount": 990.0, "bonus_earned": 9.9}
  ],
  "customer": {
    "telegram_id": 123456789,
    "id": 42,
    "card_id": "0001234567",
    "one_c_guid": null,
    "qr_code": "...",
    "bonus_balance": 1234.50,
    "referrer_telegram_id": null
  },
  "totals": {"total_amount": 990.0, "discount_total": 0.0, "bonus_spent": 0.0, "bonus_earned": 9.9}
}
```

Ошибки: `invalid_json`, `missing_field`, `invalid_payload`, `duplicate_receipt_line` (409), `missing_idempotency_key`, `invalid_idempotency_key`, `unknown_customer`, `guest_user_not_configured` (500).

## 4. Синхронизация клиента (lookup + update по card_id)

`POST /onec/customer` — поиск клиента по `card_id` и (опционально) обновление баланса/реферера/`one_c_guid`.

Поле `card_id` — обязательное. Поля `bonus_balance`, `one_c_guid`, `referrer_telegram_id` — опциональные. Если хотя бы одно из них передано — запись помечается как «write» (`status="ok"`), иначе — `status="lookup"`.

```bsl
Функция СинхронизироватьКлиента(CardID, GUID1С = "", БонусныйБаланс = Неопределено, РеферерTelegramID = 0) Экспорт

    ТелоЗапроса = Новый Структура;
    ТелоЗапроса.Вставить("card_id", CardID);

    Если НЕ ПустаяСтрока(GUID1С) Тогда
        ТелоЗапроса.Вставить("one_c_guid", GUID1С);
    КонецЕсли;
    Если БонусныйБаланс <> Неопределено Тогда
        ТелоЗапроса.Вставить("bonus_balance", Формат(БонусныйБаланс, "ЧРД=.; ЧН=0.00; ЧГ="));
    КонецЕсли;
    Если РеферерTelegramID > 0 Тогда
        ТелоЗапроса.Вставить("referrer_telegram_id", РеферерTelegramID);
    КонецЕсли;

    Попытка
        Результат = ВыполнитьЗапрос("POST", "/onec/customer", СформироватьJSON(ТелоЗапроса));
    Исключение
        Сообщить("Ошибка синхронизации клиента: " + ОписаниеОшибки());
        Возврат Неопределено;
    КонецПопытки;

    Если Результат.КодОтвета = 200 Тогда
        Возврат РазобратьJSON(Результат.Тело);
    Иначе
        Сообщить("Ошибка синхронизации клиента: HTTP " + Результат.КодОтвета + " — " + Результат.Тело);
        Возврат Неопределено;
    КонецЕсли;

КонецФункции
```

Ошибки: `400 empty_body`, `400 invalid_json`, `400 {"card_id": [...]}`, `404 customer_not_found`.

## 5. Быстрая проверка клиента

`POST /onec/customer-check` — узнать, есть ли в Lakshmi клиент по `card_id` или `telegram_id` (хотя бы одно из полей обязательно).

```bsl
Функция ПроверитьКлиента(CardID = "", TelegramID = 0) Экспорт

    ТелоЗапроса = Новый Структура;
    Если НЕ ПустаяСтрока(CardID) Тогда
        ТелоЗапроса.Вставить("card_id", CardID);
    КонецЕсли;
    Если TelegramID > 0 Тогда
        ТелоЗапроса.Вставить("telegram_id", TelegramID);
    КонецЕсли;

    Результат = ВыполнитьЗапрос("POST", "/onec/customer-check", СформироватьJSON(ТелоЗапроса));
    Если Результат.КодОтвета = 200 Тогда
        Данные = РазобратьJSON(Результат.Тело);
        Возврат Данные.card_id;   // строка card_id или "Card not found"
    КонецЕсли;
    Возврат "";

КонецФункции
```

## 7. Массовая выгрузка клиентов

`POST /onec/customers/sync` — пакетный upsert клиентов из 1С. Идентификация — по `one_c_guid`.

```bsl
// Параметры:
//   Клиенты - Массив структур:
//     .GUID1С      - Строка (<= 64)         - one_c_guid, обязательно
//     .TelegramID  - Число | Неопределено    - telegram_id, опционально
//     .Имя         - Строка                  - first_name
//     .Фамилия     - Строка                  - last_name
//     .ФИО         - Строка                  - full_name
//     .Телефон     - Строка (<= 20)
//     .Бонусы      - Число | Неопределено
//
Функция ВыгрузитьКлиентов(Клиенты) Экспорт

    Массив = Новый Массив;
    Для Каждого Стр Из Клиенты Цикл
        Эл = Новый Структура;
        Эл.Вставить("one_c_guid",  Стр.GUID1С);
        Если Стр.TelegramID <> Неопределено Тогда
            Эл.Вставить("telegram_id", Стр.TelegramID);
        КонецЕсли;
        Эл.Вставить("first_name", Стр.Имя);
        Эл.Вставить("last_name",  Стр.Фамилия);
        Эл.Вставить("full_name",  Стр.ФИО);
        Эл.Вставить("phone",      Стр.Телефон);
        Если Стр.Бонусы <> Неопределено Тогда
            Эл.Вставить("bonuses", Формат(Стр.Бонусы, "ЧРД=.; ЧН=0.00; ЧГ="));
        КонецЕсли;
        Массив.Добавить(Эл);
    КонецЦикла;

    ТелоЗапроса = Новый Структура;
    ТелоЗапроса.Вставить("customers", Массив);

    Результат = ВыполнитьЗапрос("POST", "/onec/customers/sync", СформироватьJSON(ТелоЗапроса));
    Если Результат.КодОтвета = 200 Тогда
        Возврат РазобратьJSON(Результат.Тело);   // {created, updated, errors:[{one_c_guid, reason, detail}]}
    Иначе
        Сообщить("Ошибка пакетной выгрузки клиентов: HTTP " + Результат.КодОтвета);
        Возврат Неопределено;
    КонецЕсли;

КонецФункции
```

Поведение fail-open: ошибка по отдельному клиенту попадает в массив `errors` и не отменяет батч.

## 8. Синхронизация товара

`POST /onec/product` — upsert товара по `product_code`. Категория задаётся текстом (`category`); если такой категории нет в Lakshmi, товар попадает в неразобранные.

```bsl
Функция СинхронизироватьТовар(КодТовара, Наименование, Цена, Категория, Акционный = Ложь, GUID1С = "") Экспорт

    ТелоЗапроса = Новый Структура;
    ТелоЗапроса.Вставить("product_code",   КодТовара);
    ТелоЗапроса.Вставить("name",           Наименование);
    ТелоЗапроса.Вставить("price",          Формат(Цена, "ЧРД=.; ЧН=0.00; ЧГ="));
    ТелоЗапроса.Вставить("category",       Категория);
    ТелоЗапроса.Вставить("is_promotional", Акционный);
    ТелоЗапроса.Вставить("updated_at",     Формат(ТекущаяДатаСеанса(), "ДФ=yyyy-MM-ddTHH:mm:ss"));

    Если НЕ ПустаяСтрока(GUID1С) Тогда
        ТелоЗапроса.Вставить("one_c_guid", GUID1С);
    КонецЕсли;

    Результат = ВыполнитьЗапрос("POST", "/onec/product", СформироватьJSON(ТелоЗапроса));

    Если Результат.КодОтвета = 200 ИЛИ Результат.КодОтвета = 201 Тогда
        Возврат РазобратьJSON(Результат.Тело);   // {status: "created"|"updated", product: {...}}
    Иначе
        Сообщить("Ошибка синхронизации товара: HTTP " + Результат.КодОтвета + " — " + Результат.Тело);
        Возврат Неопределено;
    КонецЕсли;

КонецФункции
```

## 9. Синхронизация дерева категорий

`POST /onec/categories/sync` — двухпроходный upsert категорий с привязкой родителей и подбором осиротевших товаров.

```bsl
// Параметры:
//   Категории - Массив структур:

        Возврат РазобратьJSON(Результат.Тело);
```

## 13. Обновление статуса заказа

`POST /onec/order/status` — обновление статуса и метаданных заказа.

Допустимые переходы:
```
new       → accepted, canceled
accepted  → assembly, canceled, new
assembly  → ready, canceled
ready     → delivery, completed, canceled
delivery  → arrived, canceled
arrived   → completed, canceled
canceled  → new
```

Поля запроса:
- `order_id` (обязательно)
- `status` (опционально; одно из VALID_STATUSES)
- `onec_guid` (опционально, привязывается при первом вызове)
- `courier_id` (опционально)
- `assembler_id` (опционально)
- `cancel_reason` (опционально, при `status="canceled"`)

```bsl
Функция ОбновитьСтатусЗаказа(IDЗаказа, НовыйСтатус, GUID1С = "", ПричинаОтмены = "", IDСборщика = 0, IDКурьера = 0) Экспорт

    ТелоЗапроса = Новый Структура;
    ТелоЗапроса.Вставить("order_id", IDЗаказа);
    Если НЕ ПустаяСтрока(НовыйСтатус) Тогда
        ТелоЗапроса.Вставить("status",   НовыйСтатус);
    КонецЕсли;
    Если НЕ ПустаяСтрока(GUID1С) Тогда
        ТелоЗапроса.Вставить("onec_guid", GUID1С);
    КонецЕсли;
    Если IDКурьера > 0 Тогда
        ТелоЗапроса.Вставить("courier_id", IDКурьера);
    КонецЕсли;
    Если IDСборщика > 0 Тогда
        ТелоЗапроса.Вставить("assembler_id", IDСборщика);
    КонецЕсли;
    Если НЕ ПустаяСтрока(ПричинаОтмены) Тогда
        ТелоЗапроса.Вставить("cancel_reason", ПричинаОтмены);
    КонецЕсли;

    Результат = ВыполнитьЗапрос("POST", "/onec/order/status", СформироватьJSON(ТелоЗапроса));
    Если Результат.КодОтвета = 200 Тогда
        Возврат РазобратьJSON(Результат.Тело);
    ИначеЕсли Результат.КодОтвета = 409 Тогда
        Сообщить("Конфликт статуса заказа #" + IDЗаказа + ": " + Результат.Тело);
        Возврат Неопределено;
    Иначе
        Сообщить("Ошибка обновления статуса: HTTP " + Результат.КодОтвета + " — " + Результат.Тело);
        Возврат Неопределено;
    КонецЕсли;

КонецФункции
```

Ошибки: `invalid_status`, `invalid_transition` (409), `already_accepted` (409), `order_not_found` (404).

При `status="canceled"` Lakshmi проставляет `canceled_by="onec"`.

`POST /onec/order/items/adjust` — пересобрать состав заказа (только в статусе `assembly`). Сервер удаляет позиции, отсутствующие в payload, и уменьшает количества (увеличение запрещено).

```bsl
// Параметры:
//   IDЗаказа - Число

Функция СкорректироватьПозицииЗаказа(IDЗаказа, Позиции) Экспорт

        Эл = Новый Структура;
        Эл.Вставить("product_code", Стр.КодТовара);
        Эл.Вставить("quantity",     Стр.Количество);
        Массив.Добавить(Эл);
    КонецЦикла;

    ТелоЗапроса = Новый Структура;
    ТелоЗапроса.Вставить("order_id", IDЗаказа);
    ТелоЗапроса.Вставить("items",    Массив);

    Результат = ВыполнитьЗапрос("POST", "/onec/order/items/adjust", СформироватьJSON(ТелоЗапроса));

        Возврат РазобратьJSON(Результат.Тело);

        Сообщить("Ошибка корректировки позиций: HTTP " + Результат.КодОтвета + " — " + Результат.Тело);

КонецФункции
```

## 15. Входящие запросы ОТ Lakshmi В 1С

Lakshmi отправляет 1С следующие запросы. Для каждого направления HTTP-сервис 1С должен принимать `POST` и проверять `X-Api-Key`.

### 15.1 Новый заказ (`ONEC_ORDER_URL`)

Заголовки:
- `X-Api-Key: {INTEGRATION_API_KEY}`
- `X-Idempotency-Key: {UUID}`
- `Content-Type: application/json`
- `Authorization: Basic …` — если в окружении заданы `ONEC_BASIC_AUTH_USER` / `ONEC_BASIC_AUTH_PASSWORD`

Тело запроса:
```json
{
  "order_id": 123,
  "status": "new",
  "created_at": "2026-04-25T10:30:00+09:00",
  "customer": {
    "id": 1,
    "card_id": "0001234567",
    "telegram_id": 123456789,
    "email": "user@example.com",
    "phone": "+79991234567",
    "full_name": "Иванов Иван"
  },
  "delivery": {
    "type": "delivery",
    "address": "ул. Пушкина, д. 10",
    "comment": "Позвонить перед доставкой",
    "product_code": "DELIVERY_ZONE_CODE"
  },
  "payment_method": "sbp",
  "payment_id": "2d4f5a6b-...",
  "payment_status": "authorized",
  "fulfillment_type": "delivery",
  "prices": {
    "products_price": "990.00",
    "delivery_price": "50.00",
    "total_price": "1040.00",
    "bonus_used": "0.00",
    "payment_amount": "1040.00"
  },
  "store_id": "store123",
  "items": [
    {"product_code": "SKU123", "name": "Молоко 1л", "quantity": 2, "price": "495.00", "line_total": "990.00"}
  ]
}
```

Ожидаемый ответ 1С (200/201):
```json
{ "onec_guid": "GUID-заказа-в-1С" }
```
Также допускается ключ `order_guid` (Lakshmi принимает оба).

Lakshmi выполняет повторные попытки: экспоненциальный backoff с jitter (20–70 сек, до `max_retries` Celery-задачи).

### 15.2 Списание/начисление бонуса (`ONEC_BONUS_URL`)

Используется при обработке чеков (fail-open).

Тело:
```json
{
  "card_id": "0001234567",
  "bonus_amount": "10.00",
  "is_accrual": true,
  "receipt_guid": "..."
}
```

Ожидаемый ответ:
```json
{ "new_balance": "1234.50" }
```

### 15.3 Синхронизация RFM-сегментов (`ONEC_RFM_SYNC_URL`)

Включается флагом `ONEC_RFM_SYNC_ENABLED=true`. Чанками по `ONEC_RFM_SYNC_CHUNK_SIZE` (по умолчанию 500) после ежемесячной фиксации тиров.

Тело — массив:
```json
[
  {"card_id": "0001234567", "segment": "Чемпионы"},
  {"card_id": "0007654321", "segment": "Постоянные"}
]
```

Ожидаемый ответ:
```json
{ "processed": 2 }
```

## Пример обработчика HTTP-сервиса 1С для приёма заказов

```bsl
Функция ПолучитьЗаказОтЛакшми(Запрос)

    КлючИзЗапроса = Запрос.Заголовки.Получить("X-Api-Key");
    Если КлючИзЗапроса <> Константы.ЛакшмиAPIKey.Получить() Тогда
        Ответ = Новый HTTPСервисОтвет(401);
        Ответ.УстановитьТелоИзСтроки("{""detail"": ""Bad API key""}");
        Возврат Ответ;
    КонецЕсли;

    ТелоСтрока = Запрос.ПолучитьТелоКакСтроку(КодировкаТекста.UTF8);
    ЧтениеJSON = Новый ЧтениеJSON;
    ЧтениеJSON.УстановитьСтроку(ТелоСтрока);
    Данные = ПрочитатьJSON(ЧтениеJSON);

    // Проверка идемпотентности — по X-Idempotency-Key
    КлючИдемпотентности = Запрос.Заголовки.Получить("X-Idempotency-Key");
    // ... поиск существующего заказа по ключу, если найден — вернуть его GUID ...

    GUIDЗаказа = СоздатьЗаказВ1С(Данные);

    РезультатJSON = Новый Структура;
    РезультатJSON.Вставить("onec_guid", GUIDЗаказа);

    ЗаписьJSON = Новый ЗаписьJSON;
    ЗаписьJSON.УстановитьСтроку();
    ЗаписатьJSON(ЗаписьJSON, РезультатJSON);

    Ответ = Новый HTTPСервисОтвет(201);
    Ответ.Заголовки.Вставить("Content-Type", "application/json");
    Ответ.УстановитьТелоИзСтроки(ЗаписьJSON.Закрыть());
    Возврат Ответ;

КонецФункции
```
